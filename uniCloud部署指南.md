# uniCloud 部署指南

## 1. 部署前准备

### 1.1 开发环境
- **HBuilderX**: 最新版本
- **uniCloud 账号**: 已注册并登录
- **微信小程序账号**: 已注册并完成认证

### 1.2 项目结构
确保项目结构如下：
```
d:\算分小程序\scoring -uni-win\
├── pages/
├── static/
├── uniCloud-tcb/
│   ├── cloudfunctions/     # 云函数
│   │   ├── user/
│   │   ├── room/
│   │   ├── score/
│   │   └── message/
│   └── database/          # 数据库
│       └── schema.json
├── App.vue
├── main.js
├── pages.json
└── package.json
```

## 2. 部署步骤

### 2.1 打开项目
1. 打开 HBuilderX
2. 点击 `文件` -> `打开目录`，选择 `d:\算分小程序\scoring -uni-win` 目录

### 2.2 初始化 uniCloud
1. 在 HBuilderX 中，右键点击 `uniCloud-tcb` 目录
2. 选择 `关联云服务空间`
3. 选择或创建一个云服务空间
4. 等待关联完成

### 2.3 部署数据库表结构
1. 右键点击 `uniCloud-tcb/database` 目录
2. 选择 `上传DB Schema`
3. 等待上传完成

### 2.4 部署云函数
1. 右键点击 `uniCloud-tcb/cloudfunctions` 目录
2. 选择 `上传所有云函数`
3. 等待上传完成

### 2.5 配置微信小程序
1. 在 `manifest.json` 文件中，配置微信小程序的 AppID
2. 确保 `uniCloud` 相关配置正确

### 2.6 构建微信小程序
1. 点击 `发行` -> `小程序-微信`
2. 填写版本号和描述
3. 点击 `发行`
4. 等待构建完成

### 2.7 上传小程序
1. 在微信开发者工具中打开构建好的项目
2. 点击 `上传` 按钮
3. 填写版本号和描述
4. 点击 `上传`

## 4. 常见问题及解决方案

### 4.1 云函数部署失败
- **原因**: 云函数代码有语法错误
- **解决方案**: 检查云函数代码，确保语法正确

### 4.2 数据库表结构上传失败
- **原因**: schema.json 文件格式错误
- **解决方案**: 检查 schema.json 文件格式，确保符合 uniCloud 要求

### 4.3 微信小程序无法登录
- **原因**: 微信小程序 AppID 配置错误
- **解决方案**: 确保在 manifest.json 中配置了正确的 AppID

### 4.4 云函数调用失败
- **原因**: 云函数权限配置错误
- **解决方案**: 检查云函数的权限配置，确保权限正确

## 6. 总结

本部署指南详细说明了如何将算分小程序从 Node.js 后端迁移到 uniCloud 云服务。通过 uniCloud，您可以无需搭建服务器和数据库，直接使用云服务来运行小程序，大大简化了部署和维护流程。

如果在部署过程中遇到问题，请参考常见问题及解决方案部分，或查阅 uniCloud 官方文档。
